# Database Patterns

## SQLx — Compile-Time Checked Queries

### Setup

```toml
[dependencies]
sqlx = { version = "0.7", features = ["runtime-tokio", "postgres", "macros", "migrate"] }
```

```rust
let pool = sqlx::postgres::PgPoolOptions::new()
    .max_connections(20)
    .acquire_timeout(Duration::from_secs(3))
    .connect(&std::env::var("DATABASE_URL")?).await?;
```

### Compile-Time Checked Queries

```rust
// query! — checks SQL at compile time, returns anonymous record
let row = sqlx::query!("SELECT id, name, email FROM users WHERE id = $1", user_id)
    .fetch_one(&pool)
    .await?;
println!("{} {}", row.id, row.name);

// query_as! — maps to a named struct
#[derive(Debug)]
struct User { id: i64, name: String, email: String }

let user = sqlx::query_as!(User, "SELECT id, name, email FROM users WHERE id = $1", user_id)
    .fetch_one(&pool)
    .await?;

// Fetch variants
.fetch_one(&pool)       // Exactly one row, error if 0 or 2+
.fetch_optional(&pool)  // Option<T>, None if no rows
.fetch_all(&pool)       // Vec<T>
.fetch(&pool)           // Stream of rows
```

### Runtime Queries (When Compile-Time Not Possible)

```rust
// Dynamic queries — no compile-time checking
let users = sqlx::query_as::<_, User>("SELECT * FROM users WHERE role = $1")
    .bind(role)
    .fetch_all(&pool)
    .await?;
```

### Migrations

```bash
# Install CLI
cargo install sqlx-cli --features postgres

# Create and run migrations
sqlx migrate add create_users      # Creates migrations/YYYYMMDD_create_users.sql
sqlx migrate run                   # Applies pending migrations
sqlx migrate revert                # Reverts last migration
```

```rust
// Embedded migrations — run at startup
sqlx::migrate!("./migrations")
    .run(&pool)
    .await?;
```

### Offline Mode (CI without Database)

```bash
# Generate query metadata for offline checking
cargo sqlx prepare                 # Creates .sqlx/ directory
# Commit .sqlx/ to VCS

# In CI, set SQLX_OFFLINE=true
SQLX_OFFLINE=true cargo check
```

## SeaORM — Convenience ORM

### Entity Definition

```rust
// entity/user.rs (typically generated with sea-orm-cli)
use sea_orm::entity::prelude::*;

#[derive(Clone, Debug, DeriveEntityModel)]
#[sea_orm(table_name = "users")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i64,
    pub name: String,
    pub email: String,
    pub created_at: DateTimeUtc,
}

#[derive(Copy, Clone, Debug, EnumIter, DeriveRelation)]
pub enum Relation {
    #[sea_orm(has_many = "super::post::Entity")]
    Posts,
}

impl Related<super::post::Entity> for Entity {
    fn to() -> RelationDef { Relation::Posts.def() }
}

impl ActiveModelBehavior for ActiveModel {}
```

### CRUD Operations

```rust
use sea_orm::{ActiveModelTrait, EntityTrait, QueryFilter, ColumnTrait, Set};

// Create
let user = user::ActiveModel {
    name: Set("Alice".to_string()),
    email: Set("alice@example.com".to_string()),
    ..Default::default()
}.insert(&db).await?;

// Read with filter
let users = User::find()
    .filter(user::Column::Name.contains("ali"))
    .order_by_asc(user::Column::CreatedAt)
    .limit(10).all(&db).await?;

// Update
let mut active: user::ActiveModel = user.into();
active.name = Set("Bob".to_string());
active.update(&db).await?;

// Eager load relations
let user_with_posts = User::find_by_id(1)
    .find_with_related(Post).all(&db).await?;
```

## Diesel — Compile-Time Query Builder

```rust
// schema.rs (auto-generated by diesel CLI)
diesel::table! {
    users (id) { id -> Int4, name -> Varchar, email -> Varchar }
}

#[derive(Queryable, Selectable)]
#[diesel(table_name = users)]
struct User { id: i32, name: String, email: String }

// Query
let results = users::table.filter(users::name.eq("Alice"))
    .limit(5).select(User::as_select()).load(&mut conn)?;
```

## Repository Pattern

```rust
#[async_trait]
pub trait UserRepository: Send + Sync {
    async fn find_by_id(&self, id: i64) -> Result<Option<User>, DbError>;
    async fn create(&self, input: CreateUser) -> Result<User, DbError>;
    async fn delete(&self, id: i64) -> Result<bool, DbError>;
}

pub struct PgUserRepository { pool: PgPool }

#[async_trait]
impl UserRepository for PgUserRepository {
    async fn find_by_id(&self, id: i64) -> Result<Option<User>, DbError> {
        sqlx::query_as!(User, "SELECT * FROM users WHERE id = $1", id)
            .fetch_optional(&self.pool).await.map_err(DbError::from)
    }
    // ...
}

// Inject as trait object — testable with mocks
async fn get_user(
    State(repo): State<Arc<dyn UserRepository>>,
    Path(id): Path<i64>,
) -> Result<Json<User>, AppError> {
    let user = repo.find_by_id(id).await?.ok_or(AppError::NotFound)?;
    Ok(Json(user))
}
```

## Transaction Handling

```rust
// SQLx transactions
let mut tx = pool.begin().await?;

let user = sqlx::query_as!(User, "INSERT INTO users (name) VALUES ($1) RETURNING *", name)
    .fetch_one(&mut *tx)
    .await?;

sqlx::query!("INSERT INTO audit_log (user_id, action) VALUES ($1, 'created')", user.id)
    .execute(&mut *tx)
    .await?;

tx.commit().await?;  // Both succeed or neither does
// If tx is dropped without commit, it auto-rollbacks
```

```rust
// SeaORM transactions
let txn = db.begin().await?;
let user = user::ActiveModel { /* ... */ }.insert(&txn).await?;
let _log = audit::ActiveModel { user_id: Set(user.id), /* ... */ }.insert(&txn).await?;
txn.commit().await?;
```

## When to Use Which

| | SQLx | SeaORM | Diesel |
|---|---|---|---|
| **Query style** | Raw SQL | Query builder + ActiveRecord | Query builder DSL |
| **Compile-time safety** | SQL checked (needs DB or offline) | Runtime | Full type-safe DSL |
| **Async** | Native | Native (built on SQLx) | Via diesel-async |
| **Learning curve** | Low (just SQL) | Medium | High (DSL, schema macro) |
| **Flexibility** | Highest | Medium | Medium |
| **Best for** | Teams comfortable with SQL | Rapid CRUD development | Strong type safety needs |

## Common Pitfalls

### Forgetting to Run Migrations

```rust
// WRONG: app starts, queries fail with "relation does not exist"
// RIGHT: run migrations at startup
sqlx::migrate!().run(&pool).await.expect("migrations failed");
```

### Pool Exhaustion

```rust
// WRONG: default pool size too small for concurrent load
let pool = PgPool::connect(url).await?;  // Default: 10 connections

// RIGHT: size pool based on expected concurrency
let pool = PgPoolOptions::new()
    .max_connections(50)
    .acquire_timeout(Duration::from_secs(3))  // Fail fast vs hang
    .connect(url).await?;
```

### N+1 Queries

```rust
// WRONG: one query per user to get their posts
let users = get_all_users(&pool).await?;
for user in &users {
    let posts = get_posts_for_user(&pool, user.id).await?;  // N queries!
}

// RIGHT: batch query
let user_ids: Vec<i64> = users.iter().map(|u| u.id).collect();
let all_posts = sqlx::query_as!(Post,
    "SELECT * FROM posts WHERE user_id = ANY($1)", &user_ids)
    .fetch_all(&pool).await?;
```

### SQLx Offline Mode Forgotten in CI

```bash
# WRONG: CI fails — no database available
# RIGHT: cargo sqlx prepare locally, commit .sqlx/, set SQLX_OFFLINE=true in CI
```

### Long-Held Transactions

```rust
// WRONG: transaction held during HTTP call — holds DB connection
let mut tx = pool.begin().await?;
let data = fetch_external_api().await?;  // Slow!
sqlx::query!("INSERT ...", data).execute(&mut *tx).await?;

// RIGHT: fetch first, then short transaction
let data = fetch_external_api().await?;
let mut tx = pool.begin().await?;
sqlx::query!("INSERT ...", data).execute(&mut *tx).await?;
tx.commit().await?;
```
